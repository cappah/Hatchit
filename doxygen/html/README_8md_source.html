16 ### Build instructions
17 
18 This is subject to change and may not be complete or entirely accurate!
19 
20 #### Linux
21 
22 All testing and development so far has been done on Ubuntu 16.04 XENIAL
23 
24 Install the following dependencies:
25 * GCC (minimum required version. 5.0) -- for C++11/14 support
26 * CMake
27 * GLFW3
28 * Bullet3
29 * Assimp
30 * TinyXML2
31 * Python 3.5 (with pip and virtualenv)
32 * Vulkan SDK (For Vulkan Support)
33 
34 Then follow these simple directions:
35 * Clone down the repo with `git clone --recursive http://github.com/thirddegree/Hatchit`
36 * Make a build dir (e.g. mkdir build)
37 * cd into build/ and run cmake ..
38 * Now just run make to build Hatchit
39 
40 #### Windows
41 
42 This is a bit more of a pain. All dependencies are submodules in the Third Party
43 directory and will need to be built before you build Hatchit.
44 
45 All testing and development has been done on Windows 10 with Visual Studio 2015
46 
47 ##### Pre-Build
48 We recommend using some sort of cmd replacement in Windows such as cmder or some sort of bash shell
49 * Install CMake and make sure it is in your path
50 * Make sure that you can run `msbuild.exe` from your shell. If it's not there try running the `vcvarsall.bat` file located at `C:\Program Files (x86)\Microsoft Visual Studio 14.0\VC\vcvarsall.bat` for Visual Studio 2015
51 * Clone down the repo with `git clone --recursive http://github.com/thirddegree/Hatchit`
52 
53 To build with Vulkan support you MUST have installed:
54 * Vulkan SDK
55 * Vulkan supported drivers!
56 
57 The build system requires that you have a working Python 3.5 installation.
58 
59 ##### Third Party
60 Next up is building all the dependencies. This should only have to be done once for your machine. After this you won't have to worry about `vcvarsall.bat` but you will need CMake.
61 
62 * Run the `setup.bat` file. This should configure AND BUILD all your dependencies
63 
64 ##### Hatchit
65 This is the easy part!
66 
67 * `cd` into `build`
68 * Open the Hatchit.sln solution file or execute it with `MSBuild.exe` to build all targets
